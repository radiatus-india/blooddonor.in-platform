# Contributing

## Before you open a pull request

- keep changes focused
- avoid adding production secrets, dumps, or environment-specific files
- prefer small, reviewable commits
- document user-facing changes in the README when relevant

## Development notes

- the current starter has no database dependency
- content lives in PHP arrays under `src/`
- shared layout rendering lives in `src/view.php`
- static assets live under `public/assets/`

## Good first contributions

- improve copy and accessibility
- add tests and linting
- build form validation and submission handlers
- add multilingual support
- improve the mobile experience for urgent request workflows

## Pull request checklist

- explain the problem clearly
- describe the approach and tradeoffs
- include screenshots for UI changes when useful
- avoid unrelated formatting churn
